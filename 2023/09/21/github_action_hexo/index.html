Github Action

GitHub Actions 是由 GitHub 推出的一项持续集成与持续部署（CI/CD）服务，使开发者和团队能够实现软件开发工作流程的自动化。通过使用 GitHub Actions，你可以直接从GitHub仓库中构建、测试和部署你的代码。这让你能够在每次向仓库提交代码时，自动执行一系列工作流程任务，包括代码的构建、测试和部署等。

开始搭建

创建Github仓库

创建两个Github仓库，一个public，另一个private。

  • private repo用来保存hexo项目的源代码（保护隐私)，使用master分支
  • public repo用来存放和发布hexo生成的静态网站，使用gh-pages分支

创建Github Token

创建一个有repo和workflow权限的GitHub Token。传送门：https://github.com/settings/tokens/new

创建repository secret

将上面生成的Token添加到pricate仓库的Secret里，随便起个名字，比如HEXO_DEPLOY。

添加Action配置文件

  1. 在private仓库的根目录下创建.github目录
  2. 在.github目录下创建workflows文件夹
  3. 在workflows文件夹下创建hexo-deploy.yml文件

使用流程

  1. 在private仓库修改hexo博客源代码（例如：增加文章、修改文章、更改主题、修改主题配置文件等等）
  2. 将本地修改push到远端仓库
    然后就会出发private仓库配置好的部署流水，执行博客生成、推送到public仓库的gh-pages分支，完成部署。
